# 任务详情展开功能设计方案

## 需求背景

当前页面只显示任务数量，无法查看具体是哪些任务。用户需要能够点击查看具体任务内容。

## 需求分析

### 核心功能
1. 点击"总任务"数字 → 展开显示所有任务列表
2. 点击"已完成任务"数字 → 展开显示已完成任务列表
3. 列表展示任务标题、状态、时间等关键信息
4. 支持折叠/收起

### 用户场景
- 用户在 Dashboard 看到任务统计数字
- 用户想了解具体有哪些任务
- 点击数字即可展开查看详情
- 再次点击或点击其他区域可收起

## UI 设计

### 布局结构
```
┌─────────────────────────────────────┐
│  总任务数: [12]                      │  ← 可点击
│  已完成任务: [8]                      │  ← 可点击
├─────────────────────────────────────┤
│  ▼ 任务详情列表 (展开时)              │
│  ┌─────────────────────────────────┐│
│  │ [进行中] 任务A - 2026-03-15    ││
│  │ [待处理] 任务B - 2026-03-14    ││
│  │ [已完成] 任务C - 2026-03-13    ││
│  └─────────────────────────────────┘│
└─────────────────────────────────────┘
```

### 交互设计
- 数字添加点击手势和 hover 效果（cursor: pointer）
- 展开时显示 ▼ 折叠图标，收起时显示 ▶
- 展开/收起动画过渡（200-300ms）
- 点击其他数字时，自动收起当前展开的列表

## 技术实现

### 数据结构
```typescript
interface Task {
  id: string;
  title: string;
  status: 'todo' | 'in-progress' | 'done';
  createdAt: string;
  updatedAt: string;
}
```

### 组件设计
1. **TaskStats**: 任务统计组件，显示总数和完成数
2. **TaskList**: 任务列表组件，支持展开/折叠

### 状态管理
```typescript
interface TaskExpandState {
  totalExpanded: boolean;
  completedExpanded: boolean;
}
```

## 接口定义

### 获取任务列表 API
```GET /api/tasks```

Response:
```json
{
  "total": 12,
  "completed": 8,
  "tasks": [
    {
      "id": "1",
      "title": "任务标题",
      "status": "todo",
      "createdAt": "2026-03-15T10:00:00Z",
      "updatedAt": "2026-03-15T10:00:00Z"
    }
  ]
}
```

## 验收标准

- [ ] 总任务数量可点击
- [ ] 已完成任务数量可点击  
- [ ] 点击后显示任务详情列表
- [ ] 可折叠收起
- [ ] 展开/收起有过渡动画
- [ ] 列表显示任务标题、状态、时间
