# OpenAPI 平台 - 功能拆解文档

## 一、功能模块总览

根据需求文档，OpenAPI 平台包含以下核心功能模块：

```
┌─────────────────────────────────────────────────────────────┐
│                      OpenAPI 平台                            │
├─────────────┬─────────────┬─────────────┬─────────────────┤
│  用户模块    │  接口管理   │  应用管理    │   权限认证模块   │
├─────────────┴─────────────┴─────────────┴─────────────────┤
│                      接口调用模块                            │
├─────────────────────────────────────────────────────────────┤
│                      管理后台（前端页面）                      │
└─────────────────────────────────────────────────────────────┘
```

## 二、详细功能拆解

## 三、数据模型设计

### 3.1 核心实体

1. **User（用户）**: 平台使用者
2. **App（应用）**: 用户创建的应用，一个用户可有多个应用
3. **Api（接口）**: 托管的接口定义
4. **ApiAccess（接口权限）**: 应用与接口的关联关系
5. **AccessToken（访问令牌）**: OAuth2.0 或 JWT 令牌
6. **ApiLog（调用日志）**: 接口调用记录

### 3.2 实体关系

```
User 1 ---- n App
App n ---- m Api (through ApiAccess)
App n ---- n AccessToken
Api n ---- n ApiLog
```

## 四、API 接口设计

### 4.1 用户接口
- `POST /api/auth/register` - 注册
- `POST /api/auth/login` - 登录
- `GET /api/user/profile` - 获取个人信息
- `PUT /api/user/profile` - 更新个人信息

### 4.2 应用接口
- `POST /api/apps` - 创建应用
- `GET /api/apps` - 应用列表
- `GET /api/apps/{id}` - 应用详情
- `PUT /api/apps/{id}` - 更新应用
- `DELETE /api/apps/{id}` - 删除应用
- `POST /api/apps/{id}/reset-secret` - 重置密钥

### 4.3 接口管理接口
- `POST /api/openapis` - 创建接口
- `GET /api/openapis` - 接口列表
- `GET /api/openapis/{id}` - 接口详情
- `PUT /api/openapis/{id}` - 更新接口
- `DELETE /api/openapis/{id}` - 删除接口
- `POST /api/openapis/{id}/publish` - 发布接口
- `POST /api/openapis/{id}/unpublish` - 下架接口

### 4.4 权限管理接口
- `POST /api/openapis/{id}/access` - 授权应用访问接口
- `DELETE /api/openapis/{id}/access/{app_id}` - 取消授权
- `GET /api/openapis/{id}/access` - 查看授权列表

### 4.5 OAuth2.0 接口
- `GET /oauth/authorize` - 授权页面
- `POST /oauth/token` - 获取令牌
- `POST /oauth/refresh` - 刷新令牌

### 4.6 开放接口（调用方使用）
- `POST /openapi/{path}` - 调用接口（网关入口）

## 五、安全考虑

1. **密码存储**: 使用 bcrypt 加密
2. **AppSecret 存储**: 哈希存储，API 调用时对比
3. **HTTPS**: 生产环境强制 HTTPS
4. **请求限流**: 防止滥用
5. **CORS**: 合理配置跨域策略

## 六、非功能需求

1. **性能**: 支持高并发接口调用
2. **可扩展**: 模块化设计，便于功能扩展
3. **易用**: 管理后台操作简单直观
4. **可维护**: 代码结构清晰，注释完善
